# 勇者 Todo：經驗與等級規則

本文件為**單一事實來源（SSOT）**，描述遊戲內等級、經驗條與能力值如何由「完成任務」推導。實作時請以 `src/hooks/useCharacter.js` 的 `calcLevelInfo` 與 `stats` 為準；**若調整規則，請同步更新本文件與程式**。

---

## 名詞

| 名詞 | 說明 |
|------|------|
| **終身完成數** | 使用者累計「完成任務」的次數，程式中為 `lifetimeCompletions`（僅用此數值計算等級，不區分任務類型）。 |
| **等級（LV）** | 整數，最小為 **1**，上限為 **120**。 |
| **經驗條（expProgress）** | 0–100 的百分比，表示**當前區間內**距離下一級的進度（見下方公式）。滿級時固定為 **100**。 |

---

## 等級區間與升級門檻

升級依「終身完成數」分為三個線性區間，每區間內**每完成固定數量的任務升一級**：

| 等級範圍 | 每升一級所需完成數 | 累計完成數門檻（進入該區間起點） |
|----------|-------------------|----------------------------------|
| **LV1 → LV5** | **3** 個任務 / 級 | **0**（從第 1 筆完成開始往上升） |
| **LV6 → LV10** | **5** 個任務 / 級 | 達到 **LV6** 需累計 **15** 筆完成（見下表） |
| **LV11 → LV120** | **10** 個任務 / 級 | 達到 **LV11** 需累計 **40** 筆完成 |

常數對照（與程式一致）：

- `TASKS_TO_LV6 = 15`：LV1–5 共 5 級 × 每級 3 任務 = **15** 筆完成時進入 LV6。
- `TASKS_TO_LV10 = 40`：LV6–10 共 5 級 × 每級 5 任務 = 再 **25** 筆，故 **15 + 25 = 40** 筆完成時進入 LV11。

---

## 計算公式（與實作對齊）

設 `c` = 終身完成數（`lifetimeCompletions`）。

### 1）`c < 15`（對應 LV1–5）

- `level = floor(c / 3) + 1`
- `expProgress = ((c % 3) / 3) * 100`

### 2）`15 ≤ c < 40`（對應 LV6–10）

- `tasksAfterLv5 = c - 15`
- `level = 6 + floor(tasksAfterLv5 / 5)`
- `expProgress = ((tasksAfterLv5 % 5) / 5) * 100`

### 3）`c ≥ 40`（對應 LV11–120）

- `tasksAfterLv10 = c - 40`
- `level = min(120, 11 + floor(tasksAfterLv10 / 10))`
- 若 `level >= 120`：`expProgress = 100`（滿級）
- 否則：`expProgress = ((tasksAfterLv10 % 10) / 10) * 100`

---

## 能力值（依等級）

基礎值 `BASE_STATS`：`atk = 5`、`def = 5`、`spd = 5`。

等級為 `level` 時：

| 屬性 | 計算方式 |
|------|----------|
| 攻擊（atk） | `value = 5 + level * 5`，`bonus = level * 5` |
| 防禦（def） | `value = 5 + level * 2`，`bonus = level * 2` |
| 速度（spd） | `value = 5 + level * 3`，`bonus = level * 3` |

（`value` 與 `bonus` 在 UI 上的用途依現有元件為準。）

## 升級回饋

- 當 `level` 較前一次計算結果**上升**時，播放升級音效（實作於 `useCharacter` 的 `useEffect`）。

## 變更流程（給維護者）

1. 先在本文件修改規則敘述與表格。
2. 再改 `calcLevelInfo`、相關常數與（若有）`stats` 公式。
3. 自測邊界：完成數 0、14→15、39→40、以及接近 LV120 的完成數。
